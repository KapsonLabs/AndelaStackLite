Emitting exit signals from a terminated process

1. When one replicated process terminates, it terminates without exit reason and kills the rest of the processes on the process stack. It doesnt seem to give an exit reason though. How do i log one? Example snippet below 

                                            payment(P) when record(P, person) ->

1. When a process terminates, it always terminates with an exit reason. The reason can be any term. A process can be set to trap exit signals by calling:

                                                process_flag(trap_exit, true)

Add this to ypur code after calling the central register server.

                                                #payment(trap_exit, true)
